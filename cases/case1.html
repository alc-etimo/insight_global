Problem · Solution · Impact

📌 Problem

  • A telecom client had 41 different ways to express "data allowance" in customer-agent interactions.
  • This caused 67% of agent searches to return zero results, increasing handling time and customer frustration.
  • No consistent taxonomy existed to classify, label, or retrieve content across systems.

✅ Solution

  • Analyzed 2M+ monthly interactions to identify semantic gaps and linguistic variation.
  • Created a controlled vocabulary with hierarchical, associative, and equivalence relations.
  • Expanded it into a 2,700-term enterprise taxonomy following ISO 25964 standards.
  • Implemented governance structures for ongoing data quality and consistency.
Impact: 67% fewer failed searches · 18% faster resolution · 30,000+ agents supported · Taxonomy adopted as standard for content classification and retrieval across the organization.

Key Insight

  • Attested Usage as Evidence: 41 variants were not "mistakes" but legitimate data points revealing the semantic field's boundaries.
  • Friction Point Identification: The gap between customer expressions, agent searches, and system classifications reveals specific placement for terminological intervention.
CRITICAL FINDING: THE AGENT'S DILEMMA

Agents developed their own intermediary vocabulary (12 search patterns) that neither matched customer expressions (41 variants) nor aligned perfectly with system classifications (3 terms). This "triple mismatch" added cognitive load and reduced efficiency.

Context: Confronted with attested variation, the task shifted from description to prescriptive normalization. The goal was to establish a preferred term around which a controlled vocabulary could be structured.
